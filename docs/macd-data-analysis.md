## 一、根因分析：为什么 165/167 合约 "转折前无MACD数据"

### 根因：合约名不匹配

**现象：**
- 诊断扫描显示日线 MACD 腿1通过率仅 **1.2%**（2/167）
- 主要原因：`转折前无MACD数据`

**实际根因：**
- `futures_macd` 表中日线 MACD 数据同时以**品种级**（`contract=EB`）和**具体合约级**（`contract=EB2609`）存储
- 品种级数据从 **2019-11-18** 起连续完整，覆盖 62 个品种
- 具体合约级数据仅覆盖其自身存续期（如 `EB2609` 从 2025-07-23 开始）
- **`scan_all_contracts()` 的 `_get_all_main_contracts()` 返回的是具体合约名**（如 `EB2609`、`A2607`）
- MACD 查询：`WHERE symbol=EB AND contract=EB2609 AND timeframe=1d AND timestamp BETWEEN [A-3d, A]`
- 如果 `EB2609` 的 MACD 数据在 Point A 时间点尚未开始 → 返回空结果 → "转折前无MACD数据"

**具体数据：**

| 分类 | 数量 | 占比 |
|------|------|------|
| 查询窗口无数据（MACD 存续期晚于 window） | 141 | 69.1% |
| MACD 开始于 Point A 之后 | 25 | 12.3% |
| 无 N 型结构 | 1 | 0.5% |

**验证：** 用品种级合约（`symbol`）代替具体合约查询 MACD 日线数据，**203/203 (100%) 都有数据**。

### 3m 数据分析

| 指标 | 数值 |
|------|------|
| 3m MACD 数据是否存在 | ✅ 存在 |
| 覆盖品种 | 51 个 |
| 时间范围 | 2026-06-08 ~ 2026-06-12（仅 5 天） |
| 存储方式 | 仅具体合约级（无品种级） |

3m 数据确实存在，但仅覆盖最近 5 天。当前不是瓶颈——先解决日线 MACD 问题后，入口更多合约升到 L1/L2，再解决 L3 的 3m 数据问题。

## 二、修复方案

**改动：** `scorer.py` 中 L1（日线）MACD 轨迹验证时，用品种级合约（`symbol`）代替具体合约（`contract`）作为 MACD 查询的 contract 参数。

**为什么要用品种级：**
- 日线 MACD 品种级数据从 2019 年起连续完整
- N 型结构 Point A/B/C 时间点无论落在何时，品种级 MACD 都覆盖
- 具体合约级 MACD 仅覆盖合约上市后的周期，无法覆盖历史 N 型结构

**不改动的地方：**
- L2（15m MACD）保持使用具体合约——15m 数据本身没有品种级存储
- L3（3m MACD）保持使用具体合约——同上

**改动文件：** `futures/scorer.py` 两个位置：
1. `evaluate()` 中 L1 `check_macd_trajectory` 调用 → `symbol` 替代 `contract` (line ~763)
2. `_check_reverse_n_structure()` 中 → `symbol` 替代 `contract` (line ~434)
