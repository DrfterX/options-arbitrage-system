## 二、核心诊断：为什么 0 ENTRY？

### 数据流分析（546 月 × 7 品种）

```
  546 月评估
    │
    ├─ 492 (90.1%) → Score=0: 无周线N型 或 日线MACD不配合
    │
    └─ 47 (8.6%) → L1通过(Score=1): 周线N型 + 日线MACD ✅
         │
         ├─ 40 (85%) → L2未通过: 小时线无N型 或 15mMACD不配合
         │
         └─ 7 (15%) → L2通过(Score=2) ✅
              │
              ├─ 7 (100%) → L3未通过: 
              │   ├─ 3m数据不足(历史月份无3m数据)
              │   └─ 无15min突破B点
              │
              └─ 0 (0%) → Score=3 (ENTRY) ❌
```

### 根本原因层次

**第一层（直接原因）：3m 数据仅 3 天**
- `check_3m_stability_replay()` 要求 `len(klines_3m) >= 30`
- 但 3m 数据只有 2026-06-10~06-12 的
- 所有历史月份检查时 `k3m` 为空，直接返回 `stable=False`
- → L3 在历史回放中**不可能通过**

**第二层（结构原因）：L1→L2 漏斗太陡**
- 周线 N 型定义使用 `SWING_WINDOWS["1w"]=2`（极窄窗口）
- 日线 MACD 要求 70% 同色（极严格）
- 15m MACD 同样要求 70% 同色
- → 三个周期同时满足条件概率极低

**第三层（设计问题）：三层并发条件过于严苛**
- 概率估算：P(周线N型+日线MACD) × P(小时N型+15mMACD|L1) × P(15mN型+3m稳定+突破|L2)
- 即使每个条件独立概率 20%：0.2 × 0.2 × 0.2 = 0.8%
- 实际更严苛：L1=8.6%, L2/L1=15%, L3/L2=0% (数据问题)
- 正常期望：每月 ~0.06 次 ENTRY（约 16 个月 1 次）
- 但系统设计目标是用大盈亏比(1:10+)覆盖低胜率

## 三、Cycle 粒度拆解

### Step 1 — 评分诊断模式（1 Cycle）

**目标：** 给 scorer.py 增加详细诊断日志，追踪每次 evaluate() 的各级失败原因

**产出物：**
- `scorer.py` 新增 `DiagnosticResult` 数据类 + 诊断模式开关
- 每级检查不通过时记录精确原因（not `stable=False` 而是 `stable=False: 3m data=12 bars < 30`）
- 运行一次 scan_all_contracts()，输出诊断报告

**依赖：** 无
**预计耗时：** 15min

### Step 2 — MACD 颜色条件调优（1-2 Cycles）

**目标：** 分析 15m MACD 的 70% 同色阈值是否过严

**产出物：**
- 从 DB 统计真实 15m MACD 颜色分布（过去 1 年）
- 输出不同阈值(50%/60%/70%/80%)下的 L2 通过率对比
- 调整 `color_tracker.py` 的阈值参数

**依赖：** Step 1
**预计耗时：** 20min

### Step 3 — 评分重置触发分析（1 Cycle）

**目标：** 调查评分重置是否过于频繁地清零有效评分

**产出物：**
- 对 replay 结果中 score=1/2 被重置的案例回溯分析
- 输出分析报告，评估是否需降低重置灵敏度

**依赖：** Step 1
**预计耗时：** 15min

### Step 4 — 3m 数据问题解决（1 Cycle）

**目标：** 确认 3m 数据修复方案

**选项 A：** 等数据自然积累（预计 30 天 → 4320 bars）
**选项 B：** 从 1m K 线合成 3m K 线（需要 1m 数据源）
**选项 C：** L3 暂时降级为 15m N 型 + 突破 + 15m MACD 稳定（不用 3m）

**产出物：** 数据修复方案决策文档

**依赖：** 无（可并行）
**预计耗时：** 10min

### Step 5 — 后端回测启动（1 Cycle）

**目标：** P0 三阶段完成后启动回测

**前提条件：** Phase ①+②+③ 全部完成
**产出物：** `scripts/phase1_backtest.py` + 回测结果

**依赖：** Step 1-4
**预计耗时：** 20min
