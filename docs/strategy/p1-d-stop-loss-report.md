# P1-D: 止损策略重构 — 纯 ATR 止损测试报告

## 测试内容
- **变更**: 移除 `Position.__init__()` 中的 C 点止损参考
- **新逻辑**: `SL = entry_price ± N × ATR`（纯 ATR 多倍止损）
- **旧逻辑**: `SL = min(c_price, entry ± 1.5×ATR)` — C 点 + ATR 混合

## 核心发现

### 1. 纯 ATR 止损无法超越 C 点参考
所有测试的 N 值（1.0~4.0）评分均低于原混合止损的 50.3。最佳纯 ATR 为 **N=3.5×（48.7）**，但仍差 1.6 分未达 WATCH 阈值。

### 2. C 点的功能不可替代
原公式 `min(c_point, 1.5×ATR)` 的 C 点提供了**自适应紧止损**：
- 当价格在 C 点附近入场 → 止损很近，单笔亏损小
- 当价格远离 C 点 → ATR 止损兜底
纯 ATR 丢失了这种 "最近极值点" 的紧止损能力。

### 3. 评分与 N 值呈 U 型曲线
- N=1.5（42.1）→ 太紧，过早止损
- N=2.5（44.9）→ 较好但仍不足
- N=3.5（48.7）→ 最佳纯 ATR
- N=4.0（48.1）→ 太宽，亏损放大

### 4. 品种层面
- **EB/TA/P** 始终为正 PnL（策略优势品种）
- **L/PG** 始终为最大亏损源（确认 P3 排除必要性）

## 结论
纯 ATR 止损替代 C 点止损**不可行** — 针对当前策略，任何倍率的纯 ATR 表现均不如原混合公式。

## 建议下一步
- **方案 A**: 回到 C 点 + ATR 混合止损，优化入场信号质量（根本性问题）
- **方案 B**: 保留 C 点止损但改为仅用于入场 ATR 计算，不直接决定 SL
- **方案 C**: C 点止损改为辅助参考（调低权重），主线仍用 ATR
