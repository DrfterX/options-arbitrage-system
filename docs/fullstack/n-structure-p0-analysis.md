## 3. 方向判定逻辑分析

### `_determine_overall_direction()` 评估

| 方面 | 分析 |
|------|------|
| 逻辑 | 比较最近 2 个 PEAK 和 2 个 TROUGH 的趋势方向 |
| 可靠 | 上升/下降趋势明显时（PEAK/TROUGH 同步向上/下），方向正确 |
| 不足 | 数据不足 4 点（<4 merged points）→ 返回 None，退回 A→B 判定 |
| 不足 | 趋势分歧（PEAK 向上但 TROUGH 向下）→ 返回 None，退回 A→B |

### 回退路径

当 `overall_direction` 为 None，由 `_determine_direction(a, b)` 根据 A→B 价格判断方向：
- B > A → LONG（正确）
- A > B → SHORT（正确）

此回退路径在多个波动方向中仍能正常工作（验证数据已证明）。

## 4. ABC 标点逻辑分析

### 前向非重叠扫描

算法：从最早极值点开始，找到第一个有效 A→B→C（满足 C1/C2/C3），跳到 C 之后继续。

**问题场景**：
1. 极值点数量不足时找不到任何结构 → IDLE（合理）
2. 合并后不足 3 个点 → IDLE（合理）
3. 无有效 ABC 匹配（如所有 C 都突破了 A）→ IDLE（合理）

### 潜在改进点

1. **C 点滑动捕捉**：`_update_c_point` 已有，但 scanning 时的 C 是最优还是第一个候选？
   - `_find_n_structure_forward` 在 C 循环中：`cond4_ok` 满足 → 立即接受（第一个满足 C4 的 C）
   - 如果没有满足 C4 → 取 `best_candidate`（第一个通过的 C）
   - **问题**：可能不是最优 C（如更极端/更近期的 C 在后面）

2. **多个有效结构的筛选**：当前取 `structures[-1]`（最后一个结构）
   - 前向扫描找到多个非重叠结构 → 取最新的一个
   - 这是合理的：最新的结构代表当前市场状态

## 5. 识别出的问题（按优先级）

### ⛔ 并非真正 bug：C4 "失败" 

213 条验证失败中，C4 占绝大部分（15m:57/57, 1h:44/44, 1d:51/51, 1w:28/61）。

C4 要求 "最新价已突破 C 点的第三笔方向" — 这是对市场状态的描述，不是对算法正确性的要求。行情未到 C 点突破位时标记为 IDLE 是合理的。

**结论**：不是算法 bug，不需要修复。

### ⚠️ 需优化：部分品种找不到有效结构

部分品种在所有周期都显示 "无有效3点结构"（如某些新品种或价格长期横盘）。这不是算法问题，但可以优化：
- 对横盘市场，极值点过于密集 → 合并后交替方向不足以形成 3 点 → IDLE
- **建议**：不修复，这是市场特征而非算法缺陷

### ✅ 前端渲染确认

需要验证前端 `futures_dashboard.html` 中的 `drawKline()` 是否正确渲染 ABC 标记：
- A/B/C 点到具体 K 线端点的映射
- 线段 A→B, B→C 的绘制
- C→最新价的方向箭头

## 6. 修改建议

### 基本上不需要重大算法修改

验证数据表明核心算法已经正确。需要关注的是：

1. **前端渲染确认**（最高优先级）：确保 K 线浮窗中 ABC 标记渲染与后端数据一致
2. **C 点选择优化**（可选）：在 `_find_n_structure_forward` 中，考虑选择更优的 C（更极端或更近期）而不是第一个有效 C
3. **无结构品种的记录**（低优先级）：记录哪些品种/周期无法找到有效结构，供人工核查
