# Skeleton Loading 加载体验优化方案

## 当前状态评估

### 已覆盖的加载优化（无需重复造轮）

| 优化点 | 状态 | 说明 |
|--------|------|------|
| SSR 初始渲染 | ✅ | 矩阵表/统计/信号卡片由 Flask 模板直接注入数据，首次加载无白屏 |
| `.skeleton` CSS 类 | ✅ | `style.css` line 770-784，shimmer 动效已定义并用于所有异步区 |
| 异步区骨架占位 | ✅ | SmartFilter、持仓、回测、健康面板均有 skeleton 占位符 |
| 5min 矩阵刷新骨架 | ✅ | `createMatrixSkeleton()` 在 fetch 期间替换 tbody |
| 加载脉冲指示 | ✅ | `mx-loading-dot` 脉冲动画（矩阵刷新时显示） |
| 数据时间戳刷新 | ✅ | `.topbar-time` 每轮自动更新 |

### 需要改进的缺陷

| # | 问题 | 严重度 | 影响面 |
|---|------|--------|--------|
| 1 | `onPageLoad` SSR 前 `_restructure_active_structures()` 阻塞渲染，DB 重时可能白屏 1-3s | 中 | 初始页面加载体验 |
| 2 | 矩阵 5min 刷新 skeleton 样式简陋：文字嵌在 skeleton 内（"Loading…"），无几何骨架 | 低 | 视觉一致性 |
| 3 | K 线弹窗加载中无骨架，只有"加载中…"文字 | 中 | K 线交互体验 |
| 4 | 异步数据加载失败全部静默处理（`/* silent */`），用户看不到错误反馈 | 中 | 可用性 |
| 5 | 所有异步区同时加载，无优先级区分 | 低 | 感知性能 |
| 6 | 信号卡片区域（SSR 渲染）无刷新骨架 | 低 | 5min 刷新一致性 |

## 设计方案

### A. 矩阵 5min 刷新骨架改进（低开销）

**现状**：`createMatrixSkeleton()` 生成带文字的 `<span class="skeleton">`

**改进**：去掉文字，改为纯几何骨架块，每行的骨架块宽度与真实数据列对齐：
- 品种列：窄矩形（40-60px）
- 4 个周期列：28×28px 方块（模拟信号色块）
- 共振列：共振条背景 + 数字位

**改动位置**：`futures_dashboard.html` 中 `createMatrixSkeleton()` 函数

### B. K 线弹窗骨架（中等开销）

**现状**：点击色块后弹窗立即出现，但图表区空白直到数据返回

**改进**：在弹窗图表区添加 `skeleton` 占位，包含：
- 价格轴骨架（左侧竖线）
- K 线示意骨架（5-8 个垂直条）
- 替换"加载中…"文字

**改动位置**：`futures_dashboard.html` + `futures.css` 中 `.kp-chart-wrap` 相关

### C. 加载失败视觉反馈（低开销）

**现状**：所有异步 fetch 的 `.catch()` 都是 `/* silent */`

**改进**：将静默失败改为：
- 对关键数据（矩阵）：骨架保持，加一个重试按钮
- 对非关键数据（持仓历史/回测）：骨架保持 + 淡红色提示 "加载失败，30s 后重试"
- 对 SmartFilter：保留现有骨架

**改动位置**：`futures_dashboard.html` JS 中对应 fetch 的 catch 块

### D. SSR 预渲染期间加载指示（中开销，可选跳过）

**现状**：`_restructure_active_structures()` 在 `render_template` 前同步执行

**改进思路**：若页面首次加载时 `matrix` 为空，前端显示骨架等待 `/api/matrix`。但当前 SSR 已有数据，此项可推迟。

## 执行计划

按最小到最大价值排序：

| 步骤 | 改动 | 预期耗时 | 复杂度 |
|------|------|---------|--------|
| 1 | K 线弹窗骨架占位 | 8min | 低 |
| 2 | 矩阵刷新骨架改进（去掉文字，几何对齐） | 5min | 低 |
| 3 | 加载失败视觉反馈 | 5min | 低 |
| 4 | 部署验证 | 5min | 低 |

**总预计**：~23min（分 2 个 Cycle 完成，本 Cycle 做 Step 1-3）

## 非改动项（保留现状的理由）

- **不新增异步加载优先级队列**：当前所有异步 fetch 是并发的（~200ms 内全部完成），引入优先级队列增加复杂度且收益有限
- **不为 SSR 初始渲染添加前端骨架**：SSR 首次加载已有完整数据，加骨架反而增加 CLS（布局偏移）
- **不修改 options_dashboard.html**：Step B.3 范围声明"先期货矩阵页面"
