# Plan — P0 N 型结构检测算法修正

## 目标

修复 `futures/n_structure.py` 中 `detect_and_save()` 算法的 A/B/C 定位逻辑，使其符合人类定义的 N 型结构正确定义。

## 当前 Bug

**现状**：`detect_and_save()` 从最新极值点（C）开始，逆向找最近的 B（交替类型）→ 最近的 A（与 C 同类型）。这导致 A/B/C 随新极值点出现而滑动。

**反例**：极值点序列 `T100 → P150 → T120 → P180 → T130`
- 当前算法：A=T120, B=P180, C=T130（滑动到了错误的点）
- **正确结果**：A=T100, B=P150, C=T120（实际的 V 型结构）；P180→T130 是第三笔的延续

## 修正后的算法

1. 从最早到最新扫描合并后的极值点
2. 对每个候选 A（从早到晚）：
   - 找 A 之后第一个交替类型的极值点 → B（第一笔终点）
   - 找 B 之后第一个与 A 同类型的极值点 → C（第二笔终点）
   - 检查 C 不突破 A（LONG: C > A, SHORT: C < A）
   - 通过 → 记录为有效结构
3. 取最后一个有效结构 = 最新的 N 型结构
4. C → 最新价格 = 第三笔跟踪
