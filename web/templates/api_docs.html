StatusHub 公开数据 API

程序化访问期货 N 型结构热力图、高评分信号排行、期权 IV 百分位快照。所有端点返回 JSON，支持 CORS 跨域。

v1.0 REST JSON CORS ✓ API Key
🌐 基础 URL https://signals.drifter.indevs.in/api/v1

🔑 认证

所有端点都需要通过 X-API-Key HTTP Header 传入 API Key。未提供或无效的 Key 将返回 401 Unauthorized。

# Header 格式
X-API-Key: sk_sh_<your_32_hex_key>
💡 获取 API Key 通过 POST /api/public/request-trial 免费获取 7 天试用 Key 或联系管理员开通
curl
curl -H "X-API-Key: sk_sh_your_key_here" \
  https://signals.drifter.indevs.in/api/v1/market/heatmap

⚡ 速率限制

基于滑动窗口实现，每个 API Key 独立计数。超限返回 429 Too Many Requests，响应中包含 retry_after_seconds 字段。

⚠️ 错误处理

所有错误响应使用统一格式：{ error: { code, message }, meta: { requested_at } }

UNAUTHORIZED缺少或无效的 API Key — 检查 X-API-Key Header
RATE_LIMITED请求频率超限 — 等待 retry_after_seconds 秒后重试
INVALID_PARAMS请求参数不合法 — 查看 error.message 了解详情
NOT_FOUND请求的资源不存在
INTERNAL_ERROR服务器内部错误 — 请稍后重试或联系管理员
# 错误响应示例 (401)

  }

GET /api/v1/market/heatmap

返回按板块分类的期货信号热力图矩阵，包含每个品种的多周期 N 型结构方向、评分、级别通过情况。是信号矩阵门户页的核心数据源。

Query 参数

参数类型必填说明默认值
sector string 可选 按板块筛选，可选：有色、贵金属、黑色、能源化工、农产品、新能源 —
sort_by string 可选 排序字段：score（评分降序）或 symbol（品种代码升序） score
limit int 可选 最大返回条数（1-100） 50

响应结构

data.signals[].symbol string 品种代码，如 "AG" data.signals[].name string 品种中文名，如 "白银" data.signals[].contract string 主力合约代码 data.signals[].sector string 所属板块 data.signals[].direction string|null 方向："LONG" | "SHORT" data.signals[].score float 综合评分 (0-300+) data.signals[].levels object 级别通过：{ l1: bool, l2: bool, l3: bool } data.signals[].n_structure object|null N 型结构详情（按 timeframe 索引），含 ABC 三点价格/时间 data.summary object 汇总统计：total, long_count, short_count, sectors[]

请求示例

curl — 获取贵金属板块热力图
curl -s -H "X-API-Key: sk_sh_your_key" \
  "https://signals.drifter.indevs.in/api/v1/market/heatmap?sector=%E8%B4%B5%E9%87%91%E5%B1%9E&limit=10" \
  | jq '.data.signals[] | {symbol, name, direction, score}'
响应示例

        "symbol": "AU",
        "name": "黄金",

}

GET /api/v1/signals/top

返回评分最高的期货信号排行，支持按方向筛选和最低评分过滤。每个品种仅返回最新信号。

Query 参数

参数类型必填说明默认值
direction string 可选 方向筛选："LONG"（多头）或 "SHORT"（空头） —
min_score float 可选 最低评分阈值（≥0） 0
limit int 可选 最大返回条数（1-100） 50

响应结构

data.signals[].symbol string 品种代码 data.signals[].name string 品种中文名 data.signals[].contract string 主力合约，如 "ag2607" data.signals[].direction string "LONG" | "SHORT" data.signals[].signal_type string 信号类型，如 "N型突破" data.signals[].score float 综合评分 data.signals[].entry_price float|null 入场价格 data.signals[].stop_loss float|null 止损价格 data.signals[].take_profit float|null 止盈价格 data.signals[].detail string 信号详细说明 data.signals[].created_at string 信号生成时间

请求示例

curl — 获取评分 260+ 的多头信号
curl -s -H "X-API-Key: sk_sh_your_key" \
  "https://signals.drifter.indevs.in/api/v1/signals/top?direction=LONG&min_score=260&limit=5" \
  | jq '.data.signals[] | {name, contract, score, entry_price, stop_loss}'

GET /api/v1/options/iv-summary

返回所有期权品种的最新隐含波动率 (IV) 百分位快照、等级分类和四分位范围。用于识别 IV 处于极端高/低水平的品种。

Query 参数

参数类型必填说明默认值
days int 可选 回溯天数计算百分位（1-365） 30
sort_by string 可选 排序："percentile"（百分位降序）或 "current_iv"（IV 值降序） percentile
limit int 可选 最大返回条数（1-100） 50

响应结构

data.iv_data[].symbol string 品种代码 data.iv_data[].name string 品种中文名 data.iv_data[].current_iv float 当前 IV 值（%） data.iv_data[].percentile float|null 历史百分位 (0-1) data.iv_data[].level string IV 等级：极端高 / 偏高 / 正常 / 偏低 / 极端低 data.iv_data[].range object IV 四分位范围：{ min, p25, p50, p75, max } data.iv_data[].samples int 回溯期内的样本数量 data.level_summary object 各等级的品种计数

请求示例

curl — 获取 180 天 IV 百分位排序
curl -s -H "X-API-Key: sk_sh_your_key" \
  "https://signals.drifter.indevs.in/api/v1/options/iv-summary?days=180&sort_by=percentile&limit=10" \
  | jq '.data.iv_data[] | {name, current_iv, percentile, level}'

🔁 完整 Python 示例

使用 Python 将 API 数据整合为一个完整的交易决策简报：

Python — 交易决策简报
import requests, json

API_BASE = "https://signals.drifter.indevs.in/api/v1"
API_KEY = "sk_sh_YOUR_KEY_HERE"
HEADERS = {"X-API-Key": API_KEY}

# 1. 获取热力图摘要
heatmap = requests.get(f"{API_BASE}/market/heatmap?limit=5", headers=HEADERS).json()
summary = heatmap["data"]["summary"]
print(f"📊 信号总览: {summary['total']} 品种 | 多头 {summary['long_count']} | 空头 {summary['short_count']}")

# 2. 获取最高评分信号
top = requests.get(f"{API_BASE}/signals/top?direction=LONG&min_score=280&limit=3", headers=HEADERS).json()
for s in top["data"]["signals"]:
    print(f"  🟢 {s['name']} ({s['contract']}) | 评分 {s['score']} | 入场 {s['entry_price']}")

# 3. 检查 IV 异常
iv = requests.get(f"{API_BASE}/options/iv-summary?days=180&limit=3", headers=HEADERS).json()
for i in iv["data"]["iv_data"]:
    pct = round(i['percentile']*100, 1) if i['percentile'] else 'N/A'
    print(f"  📈 {i['name']} IV {i['current_iv']}% | 百分位 {pct}% | {i['level']}")
⚠️ API Key 有速率限制，不建议在循环中高频轮询。建议使用 limit 参数控制单次返回量，降低请求频率。
