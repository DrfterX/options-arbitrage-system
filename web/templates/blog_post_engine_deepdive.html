一、为什么需要自动识别 N 型结构？

先做一个小测试。打开文华财经或 TradingView，随便选一个期货品种：

  1. 切换到 15 分钟图 → 找最近的 N 型 → 标记 A、B、C 三点
  2. 切换到 1 小时图 → 再来一遍
  3. 切到日线 → 再来一遍
  4. 切到周线 → 再来一遍

做完这四步，已经过去 5-10 分钟了。而且不同人画的 N 型可能完全不同——A 点选在哪根 K 线，C 点是否成立，上升还是下降，全靠个人经验。

更麻烦的是，行情刷新后，之前画的结构可能已经失效了——C 点被新 K 线突破，或者方向发生了翻转。这时候你需要重新画。

如果一个系统能自动完成这件事——实时、跨周期、可验证——你就不需要在这件事上花任何时间了。

二、N 型结构的正确定义

在深入代码之前，需要把这个基础问题说清楚。因为错误的定义会导致整个算法跑偏。

一个 N 型结构由 A、B、C 三个点构成，但很多人把 A、B、C 简单地理解为「前高、前低、当前价」，这不够精确。

上升 N 型（做多信号）

价格
↑        B（第一笔终点/最高点）
↑       ↗↘
↑      ↗  ↘
↑     ↗    ↘  C（第二笔低点）—— 第三笔起点
↑    ↗        ↘
↑   ↗          ↘     ↗
↑  ↗            ↘ ↗   最新价
↑ ↗              ↗
↑ A（第一笔起点/最低点）
└─────────────────────────→ 时间

正确标点方式：

  • A 点 = 第一笔的起点，是整个 N 型的最低点
  • A → B = 第一笔上涨，B 是第一笔的终点（最高点）
  • B → C = 第二笔下跌，C 是第二笔的终点（最低点）
  • 关键约束：C 点不能低于 A 点，否则上升 N 型不成立
  • C → 最新价 = 正在形成的第三笔，突破 B 点为确认信号

下降 N 型（做空信号）

方向相反：

  • A 点 = 第一笔的起点，是整个 N 型的最高点
  • A → B = 第一笔下跌，B 是第一笔的终点（最低点）
  • B → C = 第二笔上涨，C 是第二笔的终点（最高点）
  • 约束：C 点不能高于 A 点
  • C → 最新价 = 潜在第三笔，跌破 B 点确认做空

容易踩的坑

最常见的错误是用「前高前低」来定义 A、B、C，比如：

A = 最近的最低点，B = 最近的最高点，C = 再一个最低点

这样标出来的三点之间不一定有「第一笔 → 第二笔 → 潜在第三笔」的逻辑关系。它们可能属于同一个上涨波段，或者 C 已经破掉了 A 的位置。

正确的思路是：先判断方向，再按方向标点。

上升 N 型（同时满足）：

  1. B 价格 > A 价格 ✓（第一笔上涨确立）
  2. C 价格 < B 价格 ✓（第二笔下跌确立）
  3. C 价格 > A 价格 ✓（C 不破 A，否则失效）
  4. 最新价 > C 价格 ✓（潜在第三笔向上）

下降 N 型（同时满足）：

  1. A 价格 > B 价格 ✓（第一笔下跌确立）
  2. C 价格 > B 价格 ✓（第二笔上涨确立）
  3. C 价格 < A 价格 ✓（C 不高于 A）
  4. 最新价 < C 价格 ✓（潜在第三笔向下）

三、算法核心：前向非重叠扫描

确定了定义后，我们面临的问题是：从一堆极值点中，怎么找到最近的一个有效 N 型结构？

3.1 数据预处理

首先，从 K 线数据中提取极值点（PEAK 和 TROUGH），然后用一个合并步骤处理连续的同向极值点：

def _merge_same_type(swing_points):
    """合并连续同向极值点，保留最极端值"""
    merged = []
    for sp in swing_points:
        if merged and sp["point_type"] == merged[-1]["point_type"]:
            prev = merged[-1]
            if sp["point_type"] == "TROUGH":
                if sp["price"] < prev["price"]:
                    merged[-1] = sp  # 更低 → 更极端的低点
            else:  # PEAK
                if sp["price"] > prev["price"]:
                    merged[-1] = sp  # 更高 → 更极端的高点
        else:
            merged.append(sp)
    return merged

这个处理至关重要——K 线里连续的几根阳线会产生多个 PEAK，我们不需要它们，只要最高/最低的那个。

3.2 核心算法：前向非重叠扫描

代码最核心的部分是这个函数。思路是：从最早的极值点开始，依次尝试作为 A 点，然后找 B，再找 C。找到后跳到 C 之后继续，取最后一个结构作为当前活跃的 N 型。

def _find_n_structure_forward(merged):
    """前向非重叠扫描——返回最后一个有效的 N 型结构"""
    structures = []

    i = 0
    while i < len(merged) - 2:
        a = merged[i]
        found = False

        for b_idx in range(i + 1, len(merged)):
            if merged[b_idx]["point_type"] == a["point_type"]:
                continue  # 同类型，不是 B
            b = merged[b_idx]
            direction = _determine_direction(a["price"], b["price"])

            # 上升 N → A 必须是 TROUGH（最低点）
            # 下降 N → A 必须是 PEAK（最高点）
            if (direction == "LONG" and a["point_type"] != "TROUGH") or \
               (direction == "SHORT" and a["point_type"] != "PEAK"):
                continue

            for c_idx in range(b_idx + 1, len(merged)):
                if merged[c_idx]["point_type"] != a["point_type"]:
                    continue
                c = merged[c_idx]

                if direction == "LONG" and c["price"] <= a["price"]:
                    continue
                if direction == "SHORT" and c["price"] >= a["price"]:
                    continue

                structures.append({
                    "direction": direction,
                    "a": a, "b": b, "c": c,
                })
                i = c_idx + 1  # 非重叠：跳到 C 之后
                found = True
                break

            if found:
                break

        if not found:
            i += 1

    return structures[-1] if structures else None

这段代码的关键设计有三个：

1. 非重叠约束（non-overlapping）

找到 C 后将扫描起点设为 C 之后，而不是 C 本身或 B。这防止了「第三笔内部的小波动被误认为新结构」的滑动错误。

举个例子，如果一个上升 N 型的第三笔内部出现一个小级别的下跌再上涨，那段小波动的低点容易被误判为「新的 C 点」。非重叠约束强制要求下一个结构必须从 C 之后开始，彻底避免了这个陷阱。

2. 方向判定前置

先通过 _determine_direction(A, B) 判断是上升还是下降，然后再校验 A 点类型是否匹配（上升 N 要求 A 是最低点，下降 N 要求 A 是最高点）。这个顺序很重要——错误的顺序会导致 A 点和 B 点同时放错位置。

3. 取最后一个有效结构

structures[-1] 取到的是最近的一个有效结构。这符合直觉——我们想要的是当前仍在运行中的 N 型结构，而不是几个月前已经完成的那一个。

3.3 为什么不用逆向扫描？

最开始尝试的是从最新的极值点倒推——先找最后一个 C，然后往前找 B，再往前找 A。

但这种方法有一个致命的缺陷：A 点会随新极值点的出现而不断滑动。

比如新的极值点出现后，C 的位置变了，往前找的 B 可能也变了，整条线段全部重画，用户在 UI 上看到的就是「闪烁的 N 型结构」。交易者无法信任一个不断变化的信号。

前向扫描的哲学是：一旦锁定 A → B → C，它们就固定在各自的历史位置上。只有 C 点可以随新 K 线变化而滑动，A 和 B 不动。

四、三笔状态机

用一个简单的状态机来追踪 N 型结构的生命周期：

IDLE → LEG1 → LEG2 → LEG3 → COMPLETED
状态含义
IDLE无活跃结构，等待形成
LEG1A 点已确认（第一个极值点出现）
LEG2B 点已确认（反向极值出现，方向确立）
LEG3A → B → C 完整结构形成，第三段运行中
COMPLETED结构结束（被新结构替代）

当算法从前向扫描中找到有效的 ABC 三点时，直接将状态设为 LEG3。如果 A 之前还有同类型的极值点，说明旧结构已经被新结构取代，标记为 COMPLETED。

五、C 点滑动与动态重算

行情是实时更新的，新 K 线出现后，C 点可能不是最优的极值点了。

5.1 C 点滑动更新

当新的极值点与 C 点同类型且更极端时（上升 N 的 C 是低点，新低点更低），我们更新 C 点的位置——A 和 B 不动，C 随行情滑动：

def _update_c_point(active, swing_points, db):
    """更新 C 点——最新 swing point 更极端时滑动更新"""
    latest = swing_points[-1]

    if active["direction"] == "LONG":
        c_type = "TROUGH"
        more_extreme = latest["price"] < active["point_c_price"]
    else:
        c_type = "PEAK"
        more_extreme = latest["price"] > active["point_c_price"]

    if latest["point_type"] != c_type or not more_extreme:
        return None

    # 阈值防突破检查：C 超越 A 则结构失效
    if active["direction"] == "LONG":
        if latest["price"] <= active["point_a_price"]:
            _mark_completed(active, db)
            return None

    active["point_c_price"] = latest["price"]
    active["point_c_time"] = latest["timestamp"]
    _save_n_structure(db, active)
    return active

但有一个安全网：如果 C 点的值突破了 A 点（上升 N 中 C 跌破了 A 的位置），说明原有的上升结构已经失效，我们不再滑动，而是标记为 COMPLETED。

5.2 A 点突破：结构迁移

如果 A 点本身被突破了会怎样？比如上升 N 中 A 是最低点（TROUGH），新 K 线的价格跌破了这个低点——说明原本的「A 是最低点」这个假设已经不存在了。

这时候触发动态重算（dynamic_restructure）：

old_A 被突破 → old_B 变成 new_A → 找 new_B → 找 new_C → 方向重判

逻辑很简单：原结构的 B 点成为新结构的 A 点（因为它是当前可用的最早期极值点），然后重新找 B 和 C。

如果极值点不足以形成新结构，则回退到全量重算（detect_and_save）。

六、多周期共振扫描

单个周期的 N 型结构信号价值有限。真正有价值的信号是多个周期同时指向同一个方向——我们称之为共振。

信号矩阵覆盖四个周期：

周期用途
15 分钟短线交易信号
1 小时日内趋势
1 日线中期趋势
1 周线大趋势框架

当三个或四个周期同时显示看多（上升 N 型）时，信号强度可达 3/4 或 4/4：

                (row.resonance >= 2 ? '#f59e0b' :  // 中信号 → 橙色
                                      '#64748b');  // 弱信号 → 灰色

七、实战案例：橡胶 2609 周 K 线

以橡胶 2609 合约的周线为例（2026 年数据）：

点位价格说明
A17,220这一轮上涨的起点，最低点
B18,440第一笔终点的最高点
C17,245第二笔回调的低点（高于 A，结构保持）

这条上升 N 型满足：

  • ✅ B (18,440) > A (17,220) — 第一笔上涨
  • ✅ C (17,245) < B (18,440) — 第二笔下跌
  • ✅ C (17,245) > A (17,220) — C 不破 A，结构有效
  • ✅ 最新价 > C — 正在形成第三笔

信号解读：偏多。回调没有破前低，价格有向上突破的潜力。

八、总结与思考

做对了什么

  1. 前向非重叠扫描 — 解决了逆向扫描导致的「结构闪烁」问题，A、B 点锁定后不再变化
  2. 先判方向再标点 — 避免了方向混乱导致的标点错误
  3. C 点滑动 + A 点突破迁移 — 兼顾了结构的稳定性和对行情的实时响应
  4. 周期无关的算法统一 — 同一套逻辑跑所有周期，减少心智负担

架构取舍

  • 状态机只用 5 个状态 — 极简设计，三段的生命周期清晰
  • 非重叠约束 — 牺牲了对次级波动的敏感度，换来了信号的稳定性
  • 全量重算作为安全网 — 增量更新处理不了的情况一律回退到全量重算，避免状态错乱

一些可供借鉴的工程思考

  • 日志先行：所有结构状态变更都打日志（方向/价格/时间/原因），问题出现时可以直接追溯
  • 防御性检查：C 点滑动时持续检查是否突破 A，防止静默失效
  • 开源友好：全部代码 MIT 许可，核心技术不到 400 行 Python
